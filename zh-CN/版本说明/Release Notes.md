## 版本信息

**版本号**：23.5.1

**发版日期**：2025年11月7号

**版本升级**：v23.1.2.100及以后的版本可以直接升级到当前版本

## 版本定位

YashanDB Cloud Manager V23.5 定位为长期服务版本（LTS），本次版本正式支持 YashanDB V23.5版本，运维能力全面支持YashanDB单机、分布式、共享集群三种形态，包括监控告警、巡检管理、表空间管理、权限管理、会话管理、备份恢复、锁管理、日志分析、慢SQL等模块功能。

## 特性更新

### 监控与告警管理

#### 监控大盘管理

* 支持监控大盘的新建、重命名与删除操作。
* 支持图表的创建、编辑、删除，并提供单选与多选两种操作方式，可查看图表名称、描述、监控指标及类型等详细信息。
* 提供图表分组功能，便于用户将关联图表归入同一分组，实现统一管理与分析。

#### 监控能力增强

* 支持对YAC集群CR统计信息、数据库进程与线程数、会话连接方式数量、实例归档大小趋势、进程文件描述符数量进行监控。
* 支持监控具体等待事件，并展示事件详情。
* 支持计算服务器硬件指标监控，涵盖服务器厂商、电源、风扇、网卡、存储（NVME）、CPU及内存等信息。
* 支持YFS硬件监控，提供磁盘总览与详情页面，展示磁盘组名称、类型、总空间、剩余空间、AU Size、冗余度、状态、路径、使用率及IOPS等数据。
* 新增监控指标项： open files、 max user processes、max memory size、stack size、VM使用率等。

#### 告警功能优化

* 告警通知自定义脚本新增IP字段。
* 告警列表显示优化，支持用户自定义监控类型与时间类型，默认隐藏上次告警信息。
* 调整告警列表默认展示顺序，支持查看告警ID、对象、内容、建议、发生时间、持续时间、等级、状态及操作等详细信息。
* 默认告警策略支持自定义移除数据库实例对象，并对剩余对象进行告警。
* 创建告警项时新增描述示例Labels。
* 支持实时变更告警屏蔽状态，并记录屏蔽时间轴。
* 短信与邮件告警模板支持自定义配置，告警通知内容可灵活编排。

### 资源与SQL管理

#### 资源监控管理页面优化

* 资源监控页面整合。将原有分散的主机监控、YashanDB监控、TOPSOL详情页面合并至统一的资源监控界面，实现集中管理与查看。
* 监控维度细化：YashanDB监控从性能、SQL、存储与缓存、数据库进程四个维度展开。主机监控涵盖性能与主机进程两个维度。

#### SQL诊断与优化功能

* SQL诊断优化：支持对索引推荐类型的SQL语句进行诊断，并展示优化建议列表。
* 智能优化建议：当SQL执行时间超过设定阈值时，系统自动显示优化对象、优化类型、响应时间占比、诊断理由及具体优化建议。

#### SQL Outline管理

* Outline信息展示：提供SQL Outline相关信息的可视化界面。
* 支持Outline的创建、编辑、删除、启用/禁用、绑定与解绑操作，并支持查看近30天内的绑定记录。

### 多租户管理

* 支持CDB与PDB数据库登录，可托管多租户模式数据库。
* 提供租户列表展示，支持PDB的新增、删除、启动、停止与重启操作。
* 实现对CDB与PDB的资源监控。
* 支持CDB与PDB用户与角色管理、会话管理、数据库审计、可靠性方案、性能报告与快照管理、锁与事务分析、慢SQL分析等功能。
* 提供多租户模式拓扑图展示，增强可视化效果。

### 备份恢复增强

* 支持将数据库备份与归档日志备份通过tar格式打包为单一文件，便于用户集中查看与管理。
* 在分布式、共享集群及级联备模式下，支持恢复时自定义还原路径，简化操作流程。
* 常用操作可视化：支持归档日志（Archive Log）备份、单机数据库恢复时指定还原路径、单机数据库表空间备份与恢复的可视化操作，降低运维复杂度。
* 支持数据库恢复前的磁盘空间预检查，确保操作可靠性。
* 允许取消执行中或未执行的备份任务，进一步提升系统易用性。
* 支持备份任务失败告警，提升运维能力。

### 集群与高可用管理

* 支持ECS扩缩容。
* 适配分布式内存库架构。
* 支持托管独立级联备节点，支持在复制组主备切换时可选关闭自选举模式。
* 支持集群数据库扩容可视化功能，提升对共享集群的运维效率。
* 支持数据库滚动升级可视化，适配高可用部署架构的本地升级与一键升级。

### 用户体验优化

* 菜单栏父页面相关列表支持在浏览器会话级别保存筛选条件。
* 驾驶舱大屏支持展示数据库中文别名。
* 支持监控非ycm-agent用户下的数据库进程。
* 支持修改YASOM用户密码，并提供权限降级分配功能。
* 支持配置浏览器不保存密码。
* 实现agent的白屏化重装与重启操作。

## 规格配置限制

### 纳管数据库要求

支持纳管最多10套数据库，数据库的部署规格和形态不限制。

### 服务器配置

|  托管数据库规模| 服务器最低配置|
| -------------- | ----------------------- |
| 低于10实例     | 4核，4G内存，50G硬盘    |
| 10-100实例     | 8核，16G内存，100G硬盘  |
| 100实例以上    | 16核，32G内存，500G硬盘 |

### 操作系统要求

|  服务器类型| 操作系统| 支持版本|
| ----------- | -------- | --------------------------------------- |
| x86_64      | CentOS   | 7.9.2009                                |
| x86_64      | Ubuntu   | 20.04.1                                 |
| ARM aarch64 | 中标麒麟 | Kylin Linux Advanced Server V10 (Lance) |

### 浏览器要求

客户端支持Chrome、Firefox、Microsoft Edge浏览器方式进行访问，建议使用当前较新的版本。

|  浏览器| 支持版本|
| -------------- | ------------- |
| Google Chrome  | 88 及以上版本 |
| Firefox        | 78 及以上版本 |
| Microsoft Edge | 88 及以上版本 |

## 版本配套关系

YashanDB Cloud Manager v23.5全平台功能已适配YashanDB数据库v22.2、v23.1、v23.2、v23.3、v23.4、v23.5版本，除v23.5多租户模式未完全适配以外，其他部署形态均支持管理。
