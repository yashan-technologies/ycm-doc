当平台后端数据库选用为YashanDB时，直接修改后端数据库用户的密码可能会导致用户被锁住，**请按照本文档指引进行修改**。

## 注意事项

修改后端数据库用户的密码需要停止服务进程，**期间管理平台不可用**，请合理安排操作时间。

## 操作步骤

1. 以平台安装用户登录管理平台的每台服务器，执行如下命令停止平台服务YCM进程。

    ```shell
    # /opt/ycm为管理平台安装路径示例，请使用实际路径
    $ /opt/ycm/ycm/monit/monitctl run stop -g ycm
    ```

2. 以数据库安装用户登录后端数据库的服务器，连接后端数据库，执行如下SQL语句修改后端数据库的用户密码。

    ```shell
    # yasman/password@192.168.1.2:1688为后端数据库连接信息示例，请使用实际的连接信息
    $ yasql yasman/password@192.168.1.2:1688
    SQL> ALTER USER yasman IDENTIFIED BY "new_password";
    SQL> exit
    ```

3. 在平台每台服务器中，使用[yasadm](../../参考指南/工具命令介绍/yasadm)工具修改ycm.yaml配置文件中的加密密码。

    ```shell
    # /opt/ycm为管理平台安装路径示例，请使用实际路径
    $ /opt/ycm/ycm/scripts/yasadm password reset -n new_password -t backend
    ```

4. 在平台每台服务器中，执行如下命令启动平台服务YCM进程。

    ```shell
    # /opt/ycm为管理平台安装路径示例，请使用实际路径
    $ /opt/ycm/ycm/monit/monitctl run start -g ycm
    ```
