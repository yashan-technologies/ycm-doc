Diagnosis optimization includes performance reports, lock analysis, transaction analysis, slow SQL analysis, log analysis, and inspection.

- [Performance Report](Performance Report): Used to collect statistics related to YashanDB performance.
- [Lock Analysis](Lock Analysis): Analyzes current database lock waits and deadlock situations.
- [Transaction Analysis](Transaction Analysis): Views and rolls back currently uncommitted transactions in the database.
- [Slow SQL Analysis](Slow SQL Analysis): Used to analyze SQL with excessive execution time in database queries.
- [Log Analysis](Log Analysis): Used to collect and analyze database logs, host control logs, and host system logs.
- [Inspection](Inspection): Initiates inspections on the database periodically as needed to diagnose existing problems.
